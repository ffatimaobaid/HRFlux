DATA FLOW:
==========

1. DOCUMENT INGESTION (Admin Portal):
   policy_docs/ → embedder.py → chunking.py → vector_store.py → db.py

2. USER QUERY PROCESSING (Chat Interface):
   User Input → agent.py → rag.py → vector_store.py + keyword_search.py → gemini_llm.py → Response

3. CONVERSATION MANAGEMENT:
   chat_app.py → db.py (logs) → context_cache.py → agent.py (context)

KEY COMPONENTS:
===============

• chat_app.py: Main Streamlit UI with login/signup, chat interface, suggestion buttons
• admin_app.py: Document upload/management interface
• agent.py: Central orchestrator handling query classification and response generation
• rag.py: RAG pipeline with hybrid retrieval (semantic + keyword search)
• gemini_llm.py: Google Gemini API integration for LLM responses
• vector_store.py: ChromaDB integration for semantic search
• db.py: SQLite database for logs, users, documents, leave requests
• embedder.py: Text extraction and embedding generation
• chunking.py: Document splitting with token awareness and overlap
• keyword_search.py: Traditional keyword-based search fallback
• context_cache.py: Recent conversation context caching
